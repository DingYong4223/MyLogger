# MyLogger Chrome 插件使用说明

## 简介

MyLogger 是一款面向开发者、测试人员和排障人员的本地日志查看工具。它可以在 Chrome 中打开本地日志文件或文件夹，支持多标签查看、过滤、搜索、上下文查看、标记、日志比较、保存结果、常用过滤管理，以及基于断点 JSON 的日志规则提取。

插件主要用于 Android、后端服务、业务客户端等场景下的日志排查。普通日志查看和筛选都在浏览器本地完成；断点日志提取属于可选能力，需要配合本地 MyLogger 后台服务使用。

## 主要功能

- 通过“文件”菜单打开本地日志、常见文本文件或文件夹。
- 支持多文件多标签查看，过滤、搜索、筛选和保存都只作用于当前可见标签页。
- 自动解析日志行中的时间、级别、Tag 和内容；无法解析的列会留空，但原始内容仍可查看。
- 支持关键字、多行 OR 条件、正则表达式和大小写敏感过滤。
- 支持按时间范围、日志级别、Tag 精确匹配筛选。
- 支持搜索当前结果，并在搜索结果窗口中继续过滤。
- 点击结果行可打开日志上下文窗口，查看目标行前后日志并高亮当前行。
- 支持行标记、标记行列表、上下标记跳转和指定行号 Jump。
- 支持在“分析”菜单中比较两个已打开日志，左右分屏独立过滤、标记和跳转。
- 双击日志行可复制当前行文本。
- 支持保存过滤结果、搜索结果和筛查结果。
- 支持常用过滤的新增、编辑、删除、拖拽排序、导入和导出。
- 支持最近 20 条历史过滤记录。
- 支持中英文界面切换。
- 支持配置备份和恢复。

## 基本使用

安装插件后，点击浏览器右上角 MyLogger 图标，再点击“打开查看器”进入主页面。可以通过顶部“文件”菜单选择“打开文本/日志”，也可以把日志文件直接拖拽到页面中。

如果需要一次查看多个文件，可以在“文件”菜单中选择“打开文件夹”。插件会遍历文件夹中的可识别文本文件，并为每个文件创建独立标签页。重复打开同一个文件时，MyLogger 会复用已有标签页，避免产生重复标签。

日志载入后会以表格展示。主表格包含行号、时间、级别、Tag 和内容列。内容默认不自动换行，超出屏幕宽度时可以横向滚动，行号列会保持固定，方便查看长日志。对于大文件，MyLogger 使用虚拟渲染来减少页面卡顿。

日志窗口底部会显示当前文件名、行数和文件大小。打开多个标签页时，顶部过滤、搜索、时间筛选、级别筛选、Tag 筛选、标记、Jump 和保存过滤结果都只对当前可见标签页生效。

## 过滤和搜索

顶部 Filter 输入框用于筛选主窗口日志。可以输入普通关键字，也可以输入多行条件；多行之间按 OR 逻辑匹配。勾选 Regex 后，每一行会作为独立正则表达式使用；勾选 Case sensitive 后会区分大小写。

Search 输入框用于在当前展示结果中查找文本。点击 Search 按钮会打开搜索结果窗口，窗口内还可以继续输入 Filter 来自动过滤当前结果。点击某条搜索结果，会打开日志上下文窗口，展示该行附近的日志。

当 Filter 输入框有内容时，右侧会显示 View 按钮。点击后会打开筛查结果窗口，左侧展示过滤规则，右侧展示当前日志中命中的相关日志，并保留行号。

如果 Filter 输入框为空，右侧按钮会显示 Import，用于打开导入窗口。导入窗口可以导入本地过滤日志，也可以选择断点 JSON 并从本地服务获取过滤规则。

## 时间、级别和 Tag

点击表头 Time 可以打开时间筛选窗口，选择起点和终点后只展示该时间范围内的日志。

点击 Level 可以选择一个或多个日志级别，例如 `V`、`D`、`I`、`W`、`E`、`F`。

点击 Tag 可以输入一个或多个 Tag，点击 Confirm 后生效。Tag 采用完全匹配，多个 Tag 之间按 OR 逻辑过滤。

## 标记、跳转和上下文

用户可以标记重要日志行，并通过“全部标记”查看所有标记行。顶部的上下箭头可以在标记行之间跳转。Jump 按钮可以输入行号并跳转到指定日志行。

在过滤结果、搜索结果或筛查结果中点击某一行，会打开日志上下文窗口。该窗口展示目标行前后日志，并高亮目标行，适合查看错误发生前后的请求、状态变化或异常堆栈。

## 日志比较

顶部“分析”菜单提供 Compare 能力。点击 Compare 后，会弹出双列选择器，左右两侧分别展示当前已打开的日志文件。选择两个不同的日志后点击 Compare，即可打开日志比较窗口。

日志比较窗口采用左右分屏展示两个文件内容。窗口默认使用较大的显示区域，并保留拖拽缩放能力。左右两侧可以独立使用：

- Filter 输入框：按关键字或多行 OR 规则过滤当前侧日志。
- Regex：将过滤条件作为正则表达式处理。
- Case sensitive：区分大小写过滤。
- 行号标记：点击某一行的行号可标记或取消标记。
- 上下箭头：在当前侧已标记行之间快速跳转。
- Jump：输入行号后跳转到当前侧对应行，并高亮该行。
- 双击行：复制当前行文本。

比较窗口内的错误提示、复制提示和 Jump 校验提示会显示在窗口底部的黑色 toast 中，不会被主窗口遮挡。关闭比较窗口时会先弹出确认，确认后才退出。

## 常用过滤和历史过滤

左侧 Common Filters 区域用于保存高频过滤规则。用户可以为一组过滤词添加标题，后续点击即可覆盖顶部 Filter 输入框并应用。常用过滤支持编辑、删除、拖拽排序、导入和导出，便于在不同设备或团队成员之间共享。

MyLogger 会记录最近使用过的过滤内容，并保留最近 20 条历史过滤。重复记录会移动到最新位置，超过 20 条时会自动删除最早记录。

## 导入和断点日志

当 Filter 输入框为空时，右侧按钮会显示 Import。点击后可以导入本地过滤日志文件，也可以选择 Android Studio 插件导出的断点 JSON 文件，并通过本地 MyLogger 后台服务提取断点对应源码行中的日志字符串。

断点日志使用流程：

1. 启动本地 MyLogger 后台服务。
2. 在插件中打开日志文件。
3. 点击 Import。
4. 确认服务地址，默认是 `http://127.0.0.1:7878/analyze`。
5. 选择断点 JSON 文件。
6. 点击 Get Breakpoint Logs。
7. 插件会把后台返回的日志字符串填入 Filter 输入框，用于筛选当前日志。

该能力主要用于 Android 项目排障。普通日志内容不会通过该功能发送给后台；后台只接收用户选择的断点 JSON 内容，并结合本地源码提取日志规则。

## 实用工具

左侧 Tools 入口会打开随插件打包的本地工具页，提供 JSON 格式化、参数处理、二维码生成等常见文本处理能力。该页面不依赖外部网页。

## 帮助窗口

右上角问号按钮可以打开 MyLogger 帮助窗口，查看各项能力的简要说明。点击帮助窗口外侧的暗色背景或点击关闭按钮，都可以关闭帮助窗口。

## 用户配置

右上角设置按钮可以打开 User Settings 页面。用户可以在这里统一管理服务地址、语言、常用过滤和历史过滤。

设置页支持备份和恢复配置。备份会导出 JSON 文件，包含服务地址、语言、常用过滤和历史过滤；恢复时选择该 JSON 文件即可重新写入配置。

## 隐私说明

MyLogger 主要处理用户本地选择的日志文件。普通打开、查看、过滤、搜索、标记、上下文查看和保存都在浏览器本地完成。

插件不会主动上传日志文件到第三方服务器。只有用户主动使用 Get Breakpoint Logs 时，插件才会把选择的断点 JSON 内容发送到用户配置的本地服务地址，默认是 `http://127.0.0.1:7878/analyze`。

插件会使用浏览器本地存储保存语言、服务地址、常用过滤和历史过滤。用户可以在设置页备份、恢复或清理相关配置。

## 常见问题

### 为什么有些日志没有时间、级别或 Tag？

日志来源不同，格式不完全一致。MyLogger 会尽量解析常见格式，无法解析的列会留空，但内容仍可正常查看和搜索。

### 为什么保存结果再次打开后前面有行号？

保存结果会保留行号，方便回溯原始位置。MyLogger 已兼容这类自带行号的文本。

### 为什么保存过滤结果按钮不可点击？

没有过滤条件时该按钮会禁用，避免误保存完整日志。输入过滤条件后即可保存过滤结果。

### 为什么获取断点日志失败？

请确认本地服务已启动、服务地址正确、断点 JSON 文件有效，并且后台服务可以访问对应项目源码。
